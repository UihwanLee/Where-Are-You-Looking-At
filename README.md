# 액션 로그라이크 + 8번 출구 프로젝트

## 🎮 게임 정의
이 게임은 **액션 로그라이크**의 반복 성장 구조와  
**8번 출구**의 관찰·판단 구조를 결합한 하이브리드 게임입니다.

플레이어는 **WASD로 이동**하며 **공격은 자동으로 발동**됩니다.  
전투 중 맵에는 다양한 **이상 현상(Anomalies)** 이 발생하며,  
플레이어는 이를 관찰하고 판단하여 생존해야 합니다.

## ⭐ 핵심 재미 요소
- **빠른 전투 루프**: 자동 공격을 기반으로 한 속도감 있는 생존 전투  
- **전략적 빌드 구성**: 상점에서 무기와 아이템을 구매·강화하며 빌드를 완성  
- **관찰·판단 중심의 플레이**: 맵에 나타나는 이상 현상을 빠르게 인지하고 대응  
- **로그라이크성**: 매 판마다 다른 빌드와 선택이 누적되어 반복 플레이 동기 부여

## 3. 시스템 상세 설명
### 3.1 플레이어 시스템
- **이동 방식:** WASD 입력 기반 실시간 이동  
- **공격 방식:** 플레이어는 직접 공격하지 않고 **자동 공격(Auto Attack)** 이 주기적으로 발동  
- **무기 연동:**  
  - 장착된 모든 무기는 플레이어를 기준으로 위치를 유지하며 따라다님  
  - 여러 개의 무기 장착이 가능하며 각 무기별 공격이 개별적으로 실행  
- **스탯 적용:** 아이템에서 부여하는 스탯 변화가 즉시 플레이어 능력치에 반영됨  

---

### 3.2 무기 시스템
- **멀티 무기 장착:** 여러 무기를 동시에 장착하여 다양한 공격 패턴 구성  
- **자동 추종:** 무기들은 항상 플레이어 위치를 기준으로 따라다님  
- **피격 판정:** 무기의 공격이 몬스터에 충돌하면 데미지가 적용됨  

---

### 3.3 몬스터 시스템
- **스폰:** 방 입장 시 몬스터 자동 생성  
- **AI 추적:** 몬스터는 플레이어를 목표로 지속적으로 이동  
- **피격 및 사망 처리:** 공격을 맞으면 HP 감소 → HP가 0이면 사망  

---

### 3.4 방(ROOM) 진행 시스템
- **현재 방 번호 표시:** 맵에 현재 방 번호 UI로 표기  
- **다음 방 진입:** 특정 조건(몬스터 처치 등) 후 다음 방으로 이동  
- **방 번호 관리:**  
  - 정답 입구 → 방 번호 증가  
  - 오답 입구 → 방 번호 0으로 초기화  

---

### 3.5 상점(STORE) 시스템
- **UI 구현:** 상점 화면에서 무기·아이템 목록 확인 가능  

#### ▸ 구매
- 구매한 아이템은 즉시 장착되지 않고 **임시 보관 창고(Temporary Storage)** 로 이동  
- 플레이어는 임시 보관 창고에서 아이템을 정리하거나 장착 여부를 선택 가능  

#### ▸ 임시 보관 창고 기능
- 구매한 아이템이 우선 저장되는 전용 공간  
- 플레이어가 보유한 아이템들을 재배치하거나 정리할 수 있음  
- 아이템 장착·해제도 이 공간을 기준으로 처리  
- **상점 종료 시, 임시 보관 창고에 남아 있는 모든 아이템은 자동으로 판매됨**

#### ▸ 판매
- 아이템을 즉시 판매 가능  
- 상점 종료 시 **임시 보관 창고 잔여 아이템 자동 판매 처리**

#### ▸ 기타 기능
- **리롤:** 상점 목록을 새롭게 갱신  
- **인벤토리 조작:**  
  - 마우스 드래그로 아이템 이동  
  - 아이템 장착 / 해제  

---

### 3.6 이상 현상(Anomaly) 시스템
- **이상 현상 발생:** 특정 방에서 랜덤 또는 조건 기반으로 비정상 패턴 발생  
- **정답 구조:**  
  - 특정 상황에서는 **아래 입구가 정답**  
  - 다른 상황에서는 **위 입구가 정답**  
- **판단 결과:**  
  - 정답 입구 → 올바른 방으로 이동하며 방 번호 증가  
  - 오답 입구 → 잘못된 방 → 방 번호 0으로 초기화  

---

### 3.7 아이템 시스템
- **아이템 장착:** 플레이어 스탯에 직접 적용  
- **아이템 해제:** 인벤토리 또는 임시 보관 창고를 통해 관리  
- **스탯 부여:**  
  - 공격력, 공속 등 다양한 성능에 영향  
  - 적용 즉시 플레이어 능력치 갱신

## 4. 개발 로드맵
## 4.1 맵 구조 변경
- 현재 단순한 방 이동 구조를 개선하여 **더 명확한 경로 구조 및 방 형태 다양화** 예정  
- 플레이어 빌드 방식은 이미 고정되어 있으므로, **맵 구성의 변화를 통해 난이도·판단 요소를 강화**하는 방향으로 개편

### 4.2 상점 시스템 개편 (상인 NPC 등장)
- 기존 UI 기반 상점을 **상인 NPC 인터랙션 방식으로 변경**  
- 플레이어가 방을 탐험하며 상인을 직접 만나고 거래하는 형태로 자연스러운 흐름 구성  
- 상점 접근 방식이 ‘메뉴 선택’에서 ‘탐험 중 만나는 이벤트’로 변경됨

### 4.3 몬스터 스폰 템포 조정
- 현재 스폰 속도 및 패턴을 조정하여 전투 템포를 최적화  
- 초반 압박감을 강화하거나, 후반 지속 난이도를 조절하는 등 **게임 흐름에 맞춘 스폰 리듬 개선** 예정
